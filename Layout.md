# Layout Guide

Each file should be laid out according to the following rules:

## File Layout (.sol)

- Pragma statements
- Import statements
- Errors
- Interfaces
- Libraries
- Contracts

## Contract Layout
- Type declarations
- State variables
- Events
- Modifiers
- Functions

## Function Layout
- Constructor
- Receive function (if exists)
- Fallback function (if exists)
- External
- Public
- Internal
- Private
- View
- Pure
